# Maintainers Guide

This workspace now supports one default icon family plus any number of collection-specific SVG packs.

## Source of truth

- The default root exports come from `dsanchez07/react-allergens` and are regenerated by `pnpm run sync:upstream`.
- Additional packs live under `packages/allergen-icons/sources/collections/<collection-id>/`.
- Each collection directory must contain:
  - `collection.json`
  - `svg/*.svg`
- Recommended `collection.json` fields:
  - `id`
  - `label`
  - `description`
  - `license`
  - `sourceUrl`

Generated outputs are written into:

- `packages/allergen-icons/src/icons/<collection-id>/`
- `packages/allergen-icons/src/svg/<collection-id>/`
- `packages/allergen-icons/src/collections/<collection-id>/`
- `packages/allergen-icons-vue/src/icons/<collection-id>/`
- `packages/allergen-icons-vue/src/collections/<collection-id>/`

## Commands

Install dependencies:

```bash
pnpm install
```

Refresh the default `react-allergens` set from the upstream checkout:

```bash
pnpm run sync:upstream
```

Refresh the checked-out Erudus icons:

```bash
pnpm run sync:erudus
```

Import any other local SVG folder as a new collection:

```bash
pnpm run import:collection -- --id custom-icons --label "Custom Icons" --source ../custom-icons/svg --license MIT --source-url https://github.com/your-org/custom-icons
```

Then verify everything:

```bash
pnpm run verify
```

## Conventions

- Use a stable kebab-case `collectionId`. It becomes part of the public npm import path.
- Keep filenames kebab-case. Filenames drive icon names, manifest entries, and generated component names.
- Keep collection SVG directories flat. The importer only copies top-level `.svg` files.
- Prefer icons that map cleanly to one primary color plus an optional secondary detail color. The current API exposes `color` and `secondaryColor`.
- Keep `viewBox` values in the source SVGs. They are preserved in the generated icon data.
- Preserve upstream licensing and attribution when adding third-party packs.

## Release

1. Add or update icons.
2. Run `pnpm run verify`.
3. Add a changeset with `pnpm changeset`.
4. Merge to `main`.
5. Let the Changesets release workflow publish when `NPM_TOKEN` is configured.
